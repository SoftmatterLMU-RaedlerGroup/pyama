\doxysubsection{Detailed Description}
Defines the metadata of a module. 


\begin{DoxyParams}{Parameters}
{\em module} & The corresponding module. May be {\ttfamily None}. None or python module\\
\hline
\end{DoxyParams}
Each builtin module consists of metadata including name, version, dependencies and functionality of the module. These metadata are stored in the class {\ttfamily \mbox{\hyperlink{classpyama_1_1modules_1_1ModuleMetadata}{Module\+Metadata}}}.

The metadata have to be set when writing an own plugin module.

The following metadata are currently supported\+:


\begin{DoxyItemize}
\item {\ttfamily name} – The human-\/readable name of the module.

It is only used for displaying. Since users can distinguish modules only by their names, the name should be unique.
\item {\ttfamily id} – A string used to identify the module.

The id must be unique among all modules. It can contain any characters and should stay invariant across multiple versions of the module. The id must not be an empty string.
\item {\ttfamily version} – The version string of the module.

It consists of digits. Subversion numbers can be appended recursively, with dots as separators.
\item {\ttfamily category} – A human-\/readable category to which the plugin belongs.

The category is used for structured display of plugins in a GUI.
\item {\ttfamily group} – Identifiers of metamodules the plugin belongs to

Groups are needed to define alternatives that have the same functionality.
\item {\ttfamily conf\+\_\+dep} – Dependencies for configuration
\item {\ttfamily run\+\_\+dep} – Dependencies for running
\item {\ttfamily conf\+\_\+ret} – Return values of configuration
\item {\ttfamily run\+\_\+ret} – Return values of configuration
\end{DoxyItemize}

The dependencies of a plugin ({\ttfamily conf\+\_\+dep} and {\ttfamily run\+\_\+dep}) are defined as\+:\+: \begin{DoxyVerb}[tuple of] tuple of ("id", [tuple of] "conf_ret", [tuple of] [(<, >) [=]] "version")
\end{DoxyVerb}
 To define a dependency of built-\/in data, use an empty string as dependency id.
